When the user types /plan [topic], create a structured work plan:

## Plan Structure

1. **Goal**: What are we trying to achieve?
2. **Context**: Current state and constraints
3. **Approach**: High-level strategy
4. **Steps**: Detailed implementation steps
5. **Risks**: Potential issues and mitigations
6. **Success Criteria**: How do we know we're done?

## Output

Save the plan to `docs/plans/[topic]-plan.md` and display a summary.

## Tips

- Break large tasks into phases
- Identify dependencies between steps
- Consider rollback strategies
- Include testing checkpoints
