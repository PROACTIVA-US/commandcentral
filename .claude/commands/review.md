When the user types /review, analyze the project and provide suggestions:

## Review Checklist

1. **Check AGENTS.md** - Is it up to date with current project state?
2. **Check for uncommitted changes** - Run `git status`
3. **Check test coverage** - Are tests passing?
4. **Check documentation** - Is README accurate?
5. **Check dependencies** - Are requirements.txt files in sync?

## Provide Suggestions For

- Missing documentation
- Potential refactoring opportunities
- Test coverage gaps
- Architecture improvements
- Workflow optimizations

Format suggestions as actionable items with priorities.
